# 頭整理くん（atama-seiri-kun）仕様書

## 1. 概要

頭の中で考えていることやタスクがごちゃごちゃした時に、それを「看板（カード）」として全部書き出し、いるもの・いらないものを仕分けることで、頭の中を整理するためのツール。

ぐるぐる思考に陥ったときや、「次、何やればいいんだっけ？」となったときに、自分と向き合うために使う。

最終的に、ポジティブなもの・本当にやらなきゃいけないことだけが残った「すっきりした頭の状態」に変えることをゴールとする。

## 2. コンセプト・思想

- **吐き出す → 捨てる → 残す** の3ステップで頭を軽くする。
- 「整理術」ではなく「自分と向き合うための儀式」。判断を強制せず、本人の手で1枚ずつ仕分けることに意味がある。
- 残ったものを見て「これだけでいいんだ」と安心できる体験を提供する。

## 3. 利用シーン

- タスクや悩みが多すぎて、何から手をつければいいか分からないとき。
- 同じことをぐるぐる考えてしまって前に進めないとき。
- 一日の始まり／終わりに、頭の中をリセットしたいとき。

## 4. 基本フロー

```
[STEP 1] 全部だす     頭の中にあるものを看板方式でとにかく羅列する
   ↓
[STEP 2] 仕分ける     1枚ずつ「いる / いらない」を判断し、いらないものはゴミ箱へ
   ↓
[STEP 3] 残す         ポジティブ・本当に必要なものだけが残った頭に変わる
```

各ステップは画面上で明確に区別され、今どのステップにいるかが分かるようにする。

### STEP 1: 全部だす（ブレインダンプ）
- ユーザーは思いついたことを次々にカードとして追加する。
- 内容の良し悪しや重複を気にせず、とにかく外に出すことを促す。
- 入力のハードルを最小にする（Enter連打でどんどん追加できる等）。

### STEP 2: 仕分ける（トリアージ）
- 出したカードを1枚ずつ見て、「いる」か「いらない」かを決める。
- 「いらない」と判断したものはゴミ箱へ捨てる。
- まだ判断できないものは保留できる（「あとで」状態）。

### STEP 3: 残す（クリアな頭）
- 「いる」と判断したカードだけが残った状態を表示する。
- これが「整理後の頭の中」。達成感・安心感を演出する。
- 残ったカードを「今やるべきこと」として確認できる。

## 5. 機能要件

### 5.1 カード（看板）
- テキストを入力してカードを追加できる。
- カードの状態（ステータス）を持つ:
  - `inbox`（出したまま・未仕分け）
  - `keep`（いる）
  - `trash`（いらない＝ゴミ箱）
- カードのテキストを編集できる。
- カードを各状態へ移動できる（仕分け操作）。

### 5.2 仕分け操作
- カードを「いる」へ移す。
- カードを「いらない（ゴミ箱）」へ捨てる。
- ゴミ箱から戻せる（誤って捨てたものを復活）。
- ゴミ箱を空にして完全に削除できる。

### 5.3 ステップ進行
- STEP 1 → 2 → 3 を進める／戻る操作ができる。
- 各ステップで見せる情報を切り替える（STEP1は入力主体、STEP2は仕分け主体、STEP3は結果表示）。

### 5.4 リセット
- すべてのカードをクリアして最初からやり直せる（確認あり）。

### 5.5 データ保存
- ブラウザの localStorage に自動保存し、再読み込みしても内容が残る。
- 外部サーバーへの送信は一切しない（プライバシー保護）。

## 6. 非機能要件 / 技術制約

- **単一HTMLファイルで完結**すること（`index.html` 1枚）。
- **外部依存なし**（CDN・ライブラリ・フォント等の外部読み込みを禁止）。HTML / CSS / JavaScript はすべてファイル内にインライン記述する。
- インターネット接続がなくても動作する（オフライン完結）。
- モダンブラウザ（Chrome / Edge / Safari / Firefox の最新版）で動作すること。
- スマホ・PC 両方で見やすいレスポンシブ対応。

## 7. UI / 画面イメージ

- 上部にステップインジケーター（1 だす → 2 仕分ける → 3 残す）。
- 中央にカードの一覧エリア。
- カード追加用の入力欄。
- STEP 2 では「いる」「ゴミ箱」へ振り分けるためのボタンまたはエリアを用意。
- 配色・トーンは、気持ちが軽くなるような落ち着いた・ポジティブな印象にする。

## 8. スコープ外（やらないこと）

- 複数ユーザー・共有・アカウント機能。
- サーバー連携、クラウド同期。
- カードの締切・リマインダー等の高度なタスク管理機能。
- カテゴリ分けやタグ付けなどの複雑な分類（シンプルさを優先）。

## 9. 受け入れ基準（Acceptance Criteria）

- [ ] `index.html` をダブルクリックするだけで、外部通信なしに起動・動作する。
- [ ] STEP 1 でカードを連続して素早く追加できる。
- [ ] STEP 2 でカードを「いる」「いらない（ゴミ箱）」に仕分けできる。
- [ ] ゴミ箱に捨てたカードを戻せる。
- [ ] STEP 3 で「いる」カードだけが表示される。
- [ ] ブラウザを再読み込みしても内容が保持される。
- [ ] 全リセットで最初の状態に戻せる。
